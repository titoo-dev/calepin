Chaque merge sur main construit une image Docker taguee avec le SHA du commit. Le tag par SHA rend un rollback aussi simple qu une commande de deploy, sans rebuild. Le pipeline lance les tests dans la meme image avant de la pousser sur le registre. Ne jamais deployer une image qui n a pas passe l etape de tests en CI. .github/workflows/deploy.yml

Staging et production tirent exactement la meme image ; seul le tag promu differe entre environnements.
