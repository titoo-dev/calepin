Every POST that charges a workspace requires a client-supplied idempotency key. A client retrying a timed-out request had charged the same workspace twice. Keys are stored for 24 hours and a repeated key returns the original response. Never process a write endpoint without checking the idempotency key store first. src/api/idempotency.ts

The key is scoped per endpoint, so the same key on two different routes does not collide.
