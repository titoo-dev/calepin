Les donnees sont partitionnees par workspace_id sur plusieurs instances Postgres. Un gros workspace faisait deriver les statistiques du planificateur de requetes pour les autres workspaces. Le routage de shard se calcule via un hash de workspace_id modulo le nombre de shards. Les jointures cross-shard ne sont pas supportees ; l agregation se fait cote application. Ne jamais lancer une migration sans mettre a jour la config du routeur de shard. src/db/shard-router.ts

Ajouter un shard demande un rebalancement hors-ligne ; pas de resharding en ligne pour l instant.
