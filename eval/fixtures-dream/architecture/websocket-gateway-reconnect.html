Reconnexion automatique avec backoff exponentiel, de 200ms jusqu a 30 secondes maximum. Les clients mobiles perdent souvent le reseau lors d un changement wifi vers 4G. Chaque connexion websocket recoit un identifiant de session unique. Le serveur envoie un ping toutes les 30 secondes pour detecter une connexion morte. Ne jamais laisser un socket ouvert sans heartbeat actif. src/gateway/socket-manager.ts

Quand la connexion tombe, le client retente avec un delai croissant jusqu a ce qu elle soit retablie, sans perdre les messages en attente grace a une file locale.
