Data is sharded by workspace_id across multiple Postgres instances. A single large workspace data volume was dominating query planner statistics for unrelated workspaces. Shard routing is computed from a hash of workspace_id modulo the shard count. Cross-shard joins are not supported; aggregation happens in the application layer. Never run a migration without updating the shard router config. src/db/shard-router.ts

Adding a shard requires an offline rebalance step; there is no online resharding yet.
