All chat messages are published to a Redis channel per conversation. Multiple server instances need to fan out messages to all connected clients without sharing process memory. Each app server subscribes to conversation channels lazily, on first client join. Message ordering is guaranteed per channel, not globally. Never block the event loop waiting on a publish acknowledgment. src/queue/broadcaster.ts

This keeps servers stateless: any instance can handle any client, since messages flow through Redis rather than in-process memory.
