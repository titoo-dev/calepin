Le pipeline CI enchaine lint, tests unitaires puis build de l image dans cet ordre. Echouer vite sur le lint evite de gaspiller du temps de build sur du code qui ne passera pas la revue. Chaque etape peut etre relancee independamment depuis l interface CI. Ne jamais sauter l etape de lint, meme pour un hotfix urgent. .github/workflows/ci.yml

Ce pipeline alimente directement le pipeline de deploiement Docker en cas de succes.
