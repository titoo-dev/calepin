Chaque instance serveur publie sur une file partagee, tous les abonnes recoivent le message. Plusieurs instances serveur doivent voir le meme message pour les clients connectes a des noeuds differents. La file utilise un pattern pub/sub avec un canal par conversation. Toujours accuser reception avant de retirer un message de la file. src/queue/broadcaster.ts

Ce mecanisme permet a un client connecte sur n importe quel noeud de recevoir les messages diffuses par un autre noeud.
