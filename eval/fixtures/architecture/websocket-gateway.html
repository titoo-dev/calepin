Reconnexion automatique avec backoff exponentiel (200ms à 30s max). Les clients mobiles perdent fréquemment le réseau (changement wifi/4G). Chaque connexion websocket a un identifiant de session unique. Le serveur envoie un ping toutes les 30 secondes pour détecter les connexions mortes. Ne jamais garder un socket ouvert sans heartbeat actif. src/gateway/socket-manager.ts

Quand la connexion tombe, le client retente avec un délai croissant jusqu'à rétablissement, sans perdre les messages en attente grâce à une file locale.
