Reconnexion automatique avec backoff exponentiel (200ms a 30s max). Les clients mobiles perdent frequemment le reseau (changement wifi/4G). Chaque connexion websocket a un identifiant de session unique. Le serveur envoie un ping toutes les 30 secondes pour detecter les connexions mortes. Ne jamais garder un socket ouvert sans heartbeat actif. src/gateway/socket-manager.ts

Quand la connexion tombe, le client retente avec un delai croissant jusqu a retablissement, sans perdre les messages en attente grace a une file locale.
